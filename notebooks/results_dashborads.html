Key Findings
2.96× faster end-to-end vs Vanilla GPU RAG (930ms) vs Vanilla LLM (2753ms). Generation dominates latency across all systems — embedding and retrieval are negligible.
60× faster embedding GPU: 2ms vs CPU: 120ms per query. Dense matrix multiplications across 384-dim vectors map perfectly to CUDA cores.
32% hallucination reduction RAG drops hallucination from 89.2% (Vanilla) to 57.2% (GPU RAG) on SQuAD. Answer grounding at 100% confirms retrieval pipeline is correct.
2.42× faster than CPU RAG GPU RAG (930ms) vs CPU RAG (2252ms). GPU embedding + batched inference drive the speedup.
Generation is the real bottleneck LLM generation = ~99% of total latency (805ms out of 930ms in GPU RAG). Embedding (2ms) and retrieval (0.3ms) are negligible. Roofline insight: system is compute-bound at generation, not retrieval.
ETL: Pandas beats GPU for text preprocessing Pandas (26.7s) outperformed CuPy GPU pipeline (33.3s) on 191k string chunks. GPU suits large numerical dataframes — not NLP string ops. Key ADS insight: profile before optimising.
