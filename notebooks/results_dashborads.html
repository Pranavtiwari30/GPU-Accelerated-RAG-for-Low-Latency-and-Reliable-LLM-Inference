Key Findings

1.41x faster end-to-end
GPU RAG (1763ms) vs Vanilla LLM (2477ms). Generation dominates latency in all systems.
60x faster embedding
GPU embedding: 2ms vs CPU embedding: 120ms per query. Massive speedup on the retrieval side.
1.91x faster than CPU RAG
GPU RAG beats CPU RAG significantly due to GPU embedding + batched inference.
Context distraction on TriviaQA
RAG hallucination (91.8%) is higher than vanilla (57.6%) because retrieved Wikipedia passages contain related but not directly answering content — a known RAG failure mode.
ETL: Pandas beats GPU for text preprocessing
Pandas (26.7s total) outperformed GPU pipeline (33.3s) on 191k string chunks. GPU transfer overhead dominates for text-heavy ETL — GPU suits large numerical dataframes, not NLP string ops. This is a key ADS insight.
GPU memory stays within limits
Max observed: 5917 MB out of 6144 MB (96.3% utilisation). System remains stable throughout all 500 queries.
